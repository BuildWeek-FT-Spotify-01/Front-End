So, How Does this app Work?

In order to provide you, the user, with the best possible experience, we have incorporated a few different technologies. In this section of our page, we will explain how these work independently and how they come together. To begin, we have to understand how the data itself comes together.

This is a density plot that outlines how the data is spread out in the dataset itself. Some data falls under what we call a "Normal Distribution", which is a fancy way of saying it looks symmetrical in the way the data "falls" onto the graph. We can also see some of this data is skewed(leaning towards) to the left and right. We will need to apply some normalization to this data to ensure even distribution across the board. Afterwards, our data is ready to be processed. Before we do that though, we want to ask the question. How does this data all work together?

Through these two graphics, you can see how it is the data we look at all comes together and also how it works independently.

How Does the Model Work?

We use a model from the sklearn python library called the NearestNeighbors model. In order to make this model run quickly and efficiently, we house our data in it's own warehouse. This allows the model to access it without taking up too much of your processor's power. It takes in your input through the app and breaks it down into the metrics you see above. We can then query into the database and create a list of Euclidian distances and index positions for songs that are close in correlation. From there, we can input the list we created above and query from the database the artist, song, and the correlation. This data is added to a DataFrame to make it look better to you, the user. more on NearestNeighbors
